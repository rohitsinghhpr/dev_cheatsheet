1. What is React?
React is an open-source JavaScript library developed by Facebook for building user interfaces. It allows developers to create reusable UI components and efficiently update the UI using a declarative approach. React uses a virtual DOM to optimize rendering performance and provides tools for managing state and side effects in components.
2. What are the advantages of using React?
The advantages of React include:
  • Component-based architecture: Components are reusable, making code more modular and easier to maintain.
  • Virtual DOM: React uses a virtual DOM to update only the parts of the UI that have changed, improving performance.
  • Declarative UI: React's declarative syntax makes it easy to describe the UI, improving readability and maintainability.
  • Unidirectional data flow: This simplifies the data flow and makes the application easier to debug.
  • Rich ecosystem: React has a strong ecosystem with tools like React Router, Redux, and a large community for support.
3. What are the limitations of React?
Some limitations of React are:
  • JSX learning curve: While JSX is powerful, it may be difficult for beginners to grasp initially.
  • SEO challenges: React is a client-side rendering library, which can present SEO challenges, though solutions like server-side rendering (SSR) or static site generators can address this.
  • Performance issues with large apps: React's performance may degrade if not properly optimized in very large applications, especially with a lot of re-renders.
  • Frequent updates: React is frequently updated, and keeping up with new features and best practices may be challenging.
4. What is useState() in React?
The useState() hook is a built-in hook in React that allows you to add state to functional components. It returns an array with two values: the current state value and a function to update it. 

            import { useState } from 'react';

            const Counter = () => {
                const [count, setCount] = useState(0);

                return (
                    <div>
                        Count: {count}
                        <button onClick={() => setCount(count + 1)}>Increment</button>
                    </div>
                );
            };
5. What are keys in React?
Keys in React are unique identifiers used to track elements in a list of components. They help React efficiently update the user interface when the list changes. Keys are necessary when rendering lists of elements to prevent unnecessary re-renders. 

            const listItems = items.map(item => (
                <li key={item.id}>{item.name}</li>
            ));

JSX (JavaScript XML) is a syntax extension for JavaScript that allows you to write HTML-like code within JavaScript. JSX is then compiled to JavaScript at runtime. It makes it easier to define React components and their structure.

7. What are the differences between functional and class components?
The differences between functional and class components are:
  • Syntax: Class components use ES6 class syntax, while functional components use plain JavaScript functions.
  • State and lifecycle methods: Class components can hold state and have lifecycle methods, while functional components need hooks (e.g., useState, useEffect) to manage state and side effects.
  • Performance: Functional components are generally simpler and more lightweight, but class components may be more appropriate for complex logic.
8. What is the virtual DOM? How does React use the virtual DOM to render the UI?
The virtual DOM (VDOM) is an in-memory representation of the actual DOM elements. React uses the virtual DOM to optimize rendering by first updating the virtual DOM and then performing a diffing algorithm to compare it with the real DOM. Only the parts of the UI that have changed are re-rendered, improving performance.

The differences between controlled and uncontrolled components are:
  • Controlled components: The form element's value is controlled by React state. Every change to the form value is handled by the component.

                const inputRef = useRef();

10. What are props in React?
Props (short for properties) are read-only data passed from a parent component to a child component. They allow you to pass dynamic values to child components and configure them. Props are immutable within the child component. 

            const Welcome = ({ name }) => <div>Hello, {name}!</div>;

            const App = () => {
                return <Welcome name="Alice" />
            };
11. Explain React state and props.
- State: State refers to data that changes over time and is specific to a component. It can be updated using the useState hook or this.setState() in class components. - Props: Props are passed from a parent component to a child component. They are immutable in the child and are used for communication between components.
12. Explain about types of side effects in React component.
Types of side effects in React include:
  • Data fetching: Making network requests to load data.
  • DOM manipulation: Interacting with the DOM directly outside of React.
  • Event listeners: Adding or removing event listeners.
  • Timers: Setting intervals or timeouts.
The useEffect hook is commonly used to handle side effects in functional components.
13. What is prop drilling in React?
Prop drilling refers to the process of passing props from a parent component to deeply nested child components. This can lead to cumbersome and difficult-to-maintain code, especially in large applications.
14. What are error boundaries?
Error boundaries are React components that catch JavaScript errors anywhere in their child component tree, log those errors, and display a fallback UI instead of crashing the component tree. 

            class ErrorBoundary extends React.Component {
                constructor(props) {
                    super(props);
                    this.state = { hasError: false };
                }

                static getDerivedStateFromError(error) {
                    return { hasError: true };
                }

                componentDidCatch(error, info) {
                    console.log(error, info);
                }

                render() {
                    if (this.state.hasError) {
                        return <

h1>Something went wrong.</h1>
                    }
                    return this.props.children;
                }
            }
15. What is React Hooks?
React Hooks are functions that let you use React features like state and lifecycle methods in functional components. Hooks were introduced in React 16.8 to make it possible to use features like state, side effects, refs, and context without writing class components.
16. Explain React Hooks.
React Hooks provide a way to use state and other React features in functional components:
  • useState: For managing state in functional components.
  • useEffect: For handling side effects like data fetching, subscriptions, or manually manipulating the DOM.
  • useRef: For accessing and interacting with DOM elements or keeping a mutable reference.
  • useContext: For consuming context in functional components.
17. What are the rules that must be followed while using React Hooks?
The rules for using React hooks are:
  • Hooks should be called at the top level of a component (not inside loops, conditions, or nested functions).
  • Hooks should only be called from React function components or custom hooks.
18. What is the use of useEffect React Hooks?
The useEffect hook allows you to perform side effects in function components, such as fetching data, updating the DOM, or subscribing to events. It is called after the component renders and can be configured to run on specific state or prop changes. 

            useEffect(() => {
                fetchData();
            }, [data]);
19. Why do React Hooks make use of refs?
React Hooks use refs (via useRef) to access and modify the DOM directly or to store mutable values that do not trigger re-renders. They can be used for things like focusing an input element or storing the previous state value.
20. What are Custom Hooks?
Custom Hooks are user-defined functions that allow you to extract reusable logic from functional components. They enable you to share stateful logic between components without changing the component hierarchy. 

            function useCounter(initialValue = 0) {
                const [count, setCount] = useState(initialValue);

                return { count, increment: () => setCount(count + 1) };
            }
