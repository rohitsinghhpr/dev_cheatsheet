What is React Router 5?
React Router 5 is a popular routing library for React that allows you to implement dynamic routing in your application. It enables navigation between different components without reloading the page, providing a seamless, single-page application (SPA) experience.
Why Use React Router?
React Router simplifies navigation in React applications. With it, you can handle dynamic routes, nested routes, and URL parameters. It helps maintain an application’s state during navigation, ensuring that users can bookmark URLs or reload pages without losing the state or context of the current view.
Core Components of React Router 5
React Router 5 includes several core components:
  • BrowserRouter: A router that uses the HTML5 history API to keep UI in sync with the URL.
  • Route: A component that renders UI when its path matches the current URL.
  • Switch: Renders the first Route or Redirect that matches the location.
  • Link: A component used to create links that navigate to different routes.
  • Redirect: Used to navigate to another route programmatically.
Setting up React Router 5
To use React Router 5, first install it using npm or yarn: 

        npm install react-router-dom@5
Then, you can import and use Router components in your app.
Code Example: Basic Routing Setup 

        import React from 'react';
        import { BrowserRouter as Router, Route, Link, Switch } from 'react-router-dom';
        
        const Home = () => <h2>Home Page</h2>;
        const About = () => <h2>About Page</h2>;
        
        function App() {
          return (
            <Router>
              <nav>
                <ul>
                  <li>
                    <Link to="/">Home</Link>
                  </li>
                  <li>
                    <Link to="/about">About</Link>
                  </li>
                </ul>
              </nav>
        
              <Switch>
                <Route path="/" exact component={Home} />
                <Route path="/about" component={About} />
              </Switch>
            </Router>
          );
        }
        
        export default App;
Route Matching
React Router uses path matching to determine which component to render. The Route component matches the URL path and renders the specified component.
- Exact matching: The exact prop ensures that a route is matched only if the URL is an exact match.
- Dynamic matching: Parameters in routes can be defined using colon syntax, such as /user/:id, where :id is a dynamic segment of the path.
Code Example: Dynamic Routes 

        import React from 'react';
        import { BrowserRouter as Router, Route, Link } from 'react-router-dom';
        
        const UserProfile = ({ match }) => <h2>User Profile: {match.params.id}</h2>;
        
        function App() {
          return (
            <Router>
              <nav>
                <ul>
                  <li><Link to="/user/1">User 1</Link></li>
                  <li><Link to="/user/2">User 2</Link></li>
                </ul>
              </nav>
        
              <Route path="/user/:id" component={UserProfile} />
            </Router>
          );
        }
        
        export default App;
Nested Routes
React Router allows for nesting routes. You can define routes within other routes, and each route can render its own set of components, allowing for deep URL structures.
Code Example: Nested Routes 

        import React from 'react';
        import { BrowserRouter as Router, Route, Link, Switch } from 'react-router-dom';
        
        const Dashboard = () => (
          <div>
            <h2>Dashboard</h2>
            <ul>
              <li><Link to="/dashboard/overview">Overview</Link></li>
              <li><Link to="/dashboard/settings">Settings</Link></li>
            </ul>
          </div>
        );
        
        const Overview = () => <h3>Overview Page</h3>;
        const Settings = () => <h3>Settings Page</h3>;
        
        function App() {
          return (
            <Router>
              <Route path="/dashboard" component={Dashboard} />
              <Switch>
                <Route path="/dashboard/overview" component={Overview} />
                <Route path="/dashboard/settings" component={Settings} />
              </Switch>
            </Router>
          );
        }
        
        export default App;
Redirects and Programmatic Navigation
React Router allows navigation to a different route programmatically using the Redirect component or using the history object.
- Redirect: Used to redirect the user to a new path.
- history.push(): Can be used to navigate without user interaction (e.g., after form submission).
Code Example: Redirect 

        import React from 'react';
        import { Redirect } from 'react-router-dom';
        
        const RedirectExample = () => {
          const [redirect, setRedirect] = React.useState(false);
        
          if (redirect) {
            return <Redirect to="/dashboard" />;
          }
        
          return <button onClick={() => setRedirect(true)}>Go to Dashboard</button>;
        };
        
        export default RedirectExample;
Best Practices for React Router 5
- Use Switch to ensure that only one route is rendered at a time.
- Make routes accessible by adding semantic HTML and ensuring the right accessibility roles.
- When dealing with complex nested routes, break components into smaller reusable pieces.
- Avoid using window.location for routing; rely on React Router’s Link and history for navigation to maintain a smooth SPA experience.
Handling 404 (Not Found) Pages in React Router 5
In React Router 5, to handle 404 pages, you can create a fallback route using the Route component with no path. It will match any undefined path and render a 404 or error page.
Code Example: Handling 404 Page 

        import React from 'react';
        import { BrowserRouter as Router, Route, Link, Switch } from 'react-router-dom';
        
        const Home = () => <h2>Home Page</h2>;
        const About = () => <h2>About Page</h2>;
        const NotFound = () => <h2>404 - Page Not Found</h2>;
        
        function App() {
          return (
            <Router>
              <nav>
                <ul>
                  <li><Link to="/">Home</Link></li>
                  <li><Link to="/about">About</Link></li>
                </ul>
              </nav>
        
              <Switch>
                <Route path="/" exact component={Home} />
                <Route path="/about" component={About} />
                <Route component={NotFound} />  
              </Switch>
            </Router>
          );
        }
        
        export default App;
Programmatically Navigating with useHistory Hook
React Router 5 introduces the useHistory hook for programmatic navigation. You can use this hook to get the history object, which allows you to navigate between routes dynamically without using a Link component.
Code Example: useHistory Hook 

        import React from 'react';
        import { useHistory } from 'react-router-dom';
        
        const NavigateButton = () => {
          const history = useHistory();
        
          const handleClick = () => {
            history.push('/about');
          };
        
          return <button onClick={handleClick}>Go to About Page</button>;
        };
        
        export default NavigateButton;
useLocation Hook
The useLocation hook allows you to access the current location object, which contains the pathname, search, and hash. This can be useful when you need to perform actions based on the URL, such as highlighting the active link.
Code Example: useLocation Hook 

        import React from 'react';
        import { useLocation } from 'react-router-dom';
        
        const LocationDisplay = () => {
          const location = useLocation();
        
          return (
            <div>
              Current URL: {location.pathname}
            </div>
          );
        };
        
        export default LocationDisplay;
useParams Hook
The useParams hook allows you to access the URL parameters defined in your routes. It is particularly useful when you have dynamic routes with variables like /user/:id and need to fetch data based on the ID.
Code Example: useParams Hook 

        import React from 'react';
        import { useParams } from 'react-router-dom';
        
        const UserProfile = () => {
          const { id } = useParams();
        
          return <h2>User Profile: {id}</h2>;
        };
        
        export default UserProfile;
Redirecting After Form Submission
Redirecting the user after a successful form submission is a common use case. You can use the useHistory hook to push a new route to the history stack and navigate the user.
Code Example: Redirect After Form Submission 

        import React, { useState } from 'react';
        import { useHistory } from 'react-router-dom';
        
        const FormComponent = () => {
          const [name, setName] = useState('');
          const history = useHistory();
        
          const handleSubmit = (e) => {
            e.preventDefault();
            // Handle form submission logic here
            history.push('/thank-you');
          };
        
          return (
            <form onSubmit={handleSubmit}>
              <input
                type="text"
                value={name}
                onChange={(e) => setName(e.target.value)}
                placeholder="Enter your name"
              />
              <button type="submit">Submit</button>
            </form>
          );
        };
        
        export default FormComponent;
Using Lazy Loading with React Router
React Router supports lazy loading of components. This feature enables components to be loaded only when they are needed, improving the performance of large applications by reducing the initial loading time.
Code Example: Lazy Loading with React Router 

        import React, { Suspense, lazy } from 'react';
        import { BrowserRouter as Router, Route, Switch } from 'react-router-dom';
        
        const Home = lazy(() => import('./Home'));
        const About = lazy(() => import('./About'));
        
        function App() {
          return (
            <Router>
              <Suspense fallback="Loading...">
                <Switch>
                  <Route path="/" exact component={Home} />
                  <Route path="/about" component={About} />
                </Switch>
              </Suspense>
            </Router>
          );
        }
        
        export default App;
Handling Query Parameters
React Router 5 allows you to handle query parameters in the URL using the useLocation hook. Query parameters are often used for filtering, sorting, or other stateful information.
Code Example: Query Parameters 

        import React from 'react';
        import { useLocation } from 'react-router-dom';
        
        const QueryExample = () => {
          const location = useLocation();
          const queryParams = new URLSearchParams(location.search);
          const searchQuery = queryParams.get('search');
        
          return (
            <div>
              Search Query: {searchQuery}
            </div>
          );
        };
        
        export default QueryExample;
