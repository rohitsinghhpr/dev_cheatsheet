Sass Variables

Sass variables are used to store reusable values like colors, font sizes, or any other CSS property value. They allow you to keep your code DRY (Don't Repeat Yourself) and consistent. Variables in Sass are declared using the $ symbol.

Usage Example:

$primary-color: #3498db;
        $font-stack: 'Helvetica, sans-serif';
        
        body {
          font-family: $font-stack;
          color: $primary-color;
        }
  • Benefits:
    • Centralized control: You can change the value in one place, and it updates across your entire project.
    • Readability: Makes your code easier to understand and maintain.
Sass Nesting

Sass allows you to nest your CSS selectors in a hierarchical way, mimicking the structure of your HTML. This makes it easier to read and maintain.

Usage Example:

nav {
          ul {
            list-style: none;
            margin: 0;
            padding: 0;
          }
        
          li {
            display: inline-block;
        
            a {
              text-decoration: none;
              color: #333;
        
              &:hover {
                color: #555;
              }
            }
          }
        }

How it compiles:

The above example compiles to:

nav ul {
          list-style: none;
          margin: 0;
          padding: 0;
        }
        
        nav li {
          display: inline-block;
        }
        
        nav li a {
          text-decoration: none;
          color: #333;
        }
        
        nav li a:hover {
          color: #555;
        }
  • Benefits:
    • Better organization: Keeps related styles together.
    • Less repetitive code.
Sass @import

The @import directive in Sass allows you to split your styles into smaller, manageable files and import them into a main file. Unlike the CSS @import, the Sass version compiles all files into a single CSS file, improving performance.

Usage Example:

// _variables.scss
        $primary-color: #3498db;
        
        // _buttons.scss
        button {
          background-color: $primary-color;
          border: none;
          color: #fff;
        }
        
        // main.scss
        @import 'variables';
        @import 'buttons';

How it compiles:

All imported files are combined into one CSS file, reducing HTTP requests.

Note: In Dart Sass, @use is now recommended instead of @import for better scoping and modularity.

Sass @mixin

Mixins allow you to create reusable blocks of code. They are similar to functions in programming and can accept arguments.

Usage Example:

@mixin border-radius($radius) {
          -webkit-border-radius: $radius;
          -moz-border-radius: $radius;
          border-radius: $radius;
        }
        
        button {
          @include border-radius(5px);
        }

How it compiles:

button {
          -webkit-border-radius: 5px;
          -moz-border-radius: 5px;
          border-radius: 5px;
        }
  • Benefits:
    • Code reusability: Write the code once, use it anywhere.
    • Parameters: Customize mixins with arguments for flexible outputs.
Sass @extend

The @extend directive allows you to share styles across selectors without duplicating code. Itâ€™s great for reducing redundancy and keeping your styles DRY.

Usage Example:

%button-style {
          padding: 10px 20px;
          border: none;
          color: #fff;
        }
        
        .button-primary {
          @extend %button-style;
          background-color: blue;
        }
        
        .button-secondary {
          @extend %button-style;
          background-color: gray;
        }

How it compiles:

.button-primary, .button-secondary {
          padding: 10px 20px;
          border: none;
          color: #fff;
        }
        
        .button-primary {
          background-color: blue;
        }
        
        .button-secondary {
          background-color: gray;
        }
  • Benefits:
    • Avoids duplication: Shares styles efficiently.
    • Maintains consistency: Ensures related styles look uniform.
Key Differences Between @mixin and @extend
Feature @mixin @extend
Usage Reusable styles with or without arguments Inherits styles from a placeholder (%)
Output Copies code wherever used Combines selectors in compiled CSS
Flexibility Highly flexible Limited to shared selectors
Sass Functions
Sass provides built-in functions to manipulate values, such as colors, strings, numbers, and more. You can also create custom functions using `@function`. 

        // Built-in example
        $lighten-color: lighten(#3498db, 20%);
        $darken-color: darken(#3498db, 20%);
        
        // Custom function
        @function double($number) {
          @return $number * 2;
        }
        
        div {
          width: double(10px);
          background-color: $lighten-color;
        }
Sass String
Sass provides string manipulation features like concatenation, string length, and substring extraction. 

        $font-family: 'Helvetica';
        $font-weight: 'bold';
        
        // String concatenation
        $font-style: #{$font-family} #{$font-weight};
        
        div {
          font: $font-style;
        }
Sass Numeric
Sass supports arithmetic operations on numbers and units, such as addition, subtraction, multiplication, and division. 

        $base-size: 16px;
        
        // Arithmetic operations
        $large-size: $base-size * 1.5;
        
        h1 {
          font-size: $large-size;
        }
Sass List
Sass lists allow you to group values. You can access, modify, and iterate through lists. 

        $colors: red, blue, green;
        
        // Accessing list values
        $first-color: nth($colors, 1);
        
        // Loop through list
        @each $color in $colors {
          .text-#{$color} {
            color: $color;
          }
        }
Sass Map
Maps are key-value pairs in Sass, useful for organizing related data. 

        $theme-colors: (
          primary: #3498db,
          secondary: #2ecc71,
          danger: #e74c3c
        );
        
        // Accessing map values
        $primary-color: map-get($theme-colors, primary);
        
        // Iterating through maps
        @each $name, $color in $theme-colors {
          .btn-#{$name} {
            background-color: $color;
          }
        }
Sass Selector
Sass provides selector functions to manipulate and query selectors. 

        .button {
          &--primary {
            color: blue;
          }
        }
        
        $nested: selector-nest('.parent', '.child');
        $append: selector-append('.button', '--secondary');
        
        // Result
        // $nested: .parent .child
        // $append: .button--secondary
Sass Introspection
Introspection allows you to query information about Sass values, like their type or existence. 

        $test: 42;
        
        // Check type
        @if type-of($test) == 'number' {
          .result {
            content: "The value is a number";
          }
        }
        
        // Check variable existence
        @if variable-exists(test) {
          .result {
            content: "Variable exists";
          }
        }
Sass Color
Sass provides many color manipulation functions, like adjusting brightness, saturation, and hue. 

        // Adjust color
        $base-color: #3498db;
        $lighter: lighten($base-color, 20%);
        $darker: darken($base-color, 20%);
        
        // Resulting styles
        div {
          background-color: $lighter;
          border-color: $darker;
        }
